## 3. 类和对象的关系

```python
class User:
    pass

u = User()
```

这里：

| 名称 | 含义 |
|---|---|
| `User` | 类，模板 |
| `u` | 对象，由类创建出来的具体实例 |

一句话：

> 类是模板，对象是根据模板创建出来的具体东西。

比如：

```text
User 类：用户模板
u1：张三这个用户
u2：李四这个用户
```

类更像设计图，对象是根据设计图造出来的具体东西。

## 4. 实例方法：self 管对象

实例方法是给**对象**用的。

```python
class User:
    def __init__(self, name):
        self.name = name

    def show_name(self):
        return self.name
```

调用：

```python
u = User("张三")
print(u.show_name())
```

这里：

```python
self.name
```

表示：

```text
当前这个对象自己的 name
```

也就是：

```text
u.name = "张三"
```

所以：

> 只要方法要操作某个具体对象的数据，就用 `self`。

### 为什么普通方法要有 self

你表面上这样调用：

```python
u.show_name()
```

Python 实际上会这样处理：

```python
User.show_name(u)
```

所以方法定义里必须写：

```python
def show_name(self):
```

这里的 `self` 就是当前对象 `u`。

如果没有 `self`，Python 自动传进来的对象就没人接住，参数数量就会对不上。

## 5. 类方法：cls 管类

类方法是给**类本身**用的。

```python
class User:
    count = 0

    @classmethod
    def get_count(cls):
        return cls.count
```

调用：

```python
User.get_count()
```

这里：

```python
cls
```

代表：

```text
User 这个类本身
```

所以：

```python
cls.count
```

就是：

```python
User.count
```

类方法适合：

```text
1. 操作类变量
2. 统计类级别信息
3. 根据字符串、字典、配置创建对象
```

---

## 6. 类方法为什么能创建对象

比如：

```python
class User:
    def __init__(self, name):
        self.name = name

    @classmethod
    def from_string(cls, text):
        return cls(text)
```

调用：

```python
u = User.from_string("张三")
```

这里：

```python
cls
```

等于：

```python
User
```

所以：

```python
return cls(text)
```

等价于：

```python
return User(text)
```

也就是创建对象。

类方法常用来做：

```text
字符串 -> 对象
字典 -> 对象
配置文件 -> 对象
环境变量 -> 对象
```

常见命名：

| 方法名 | 含义 |
|---|---|
| `from_dict()` | 从字典创建对象 |
| `from_json()` | 从 JSON 创建对象 |
| `from_config()` | 从配置创建对象 |
| `from_env()` | 从环境变量创建对象 |
| `create()` | 工厂创建 |
| `load()` | 加载并创建 |

### 为什么不用对象来创建对象

因为这时对象还没创建。

比如：

```python
u = User.from_string("张三")
```

你是先通过 `User` 类创建出 `u`。如果非要用 `u.from_string()`，那就变成“先有对象，再用对象创建对象”。

## 7. 静态方法：工具函数

静态方法只是放在类里面的普通函数。

```python
class MathTool:
    @staticmethod
    def add(a, b):
        return a + b
```

调用：

```python
MathTool.add(1, 2)
```

它只需要：

```text
a 和 b
```

不需要对象，也不需要类。

所以：

> 静态方法适合写工具函数。

比如：

```text
加法
格式校验
字符串处理
单位转换
简单数据转换
```

如果这个函数和类关系不强，直接写普通函数往往更清楚：

```python
def normalize_text(text):
    return text.strip().lower()
```

不要为了“看起来面向对象”硬塞进类里。那不是架构，是装饰性复杂。

## 8. LeetCode 中 Solution、object、self 的来源

LeetCode 常见模板：

```python
class Solution(object):
    def twoSum(self, nums, target):
        pass
```

### 8.1 为什么要有 class Solution

因为 LeetCode 平台规定要这样写。

后台大概会这样调用：

```python
obj = Solution()
result = obj.twoSum(nums, target)
```

所以你要把答案写在 `Solution` 类里面。

### 8.2 object 是什么

```python
class Solution(object):
```

`object` 是 Python 里所有类的基础父类。

在 Python 3 里，这两种基本一样：

```python
class Solution(object):
    pass
```

```python
class Solution:
    pass
```

`object` 是历史遗留写法，LeetCode 模板保留了它。自己写 Python 3 代码时，可以直接写 `class Solution:`。

### 8.3 为什么 twoSum 里面要有 self

```python
def twoSum(self, nums, target):
```

因为 `twoSum` 被写成了类里的实例方法。

LeetCode 后台调用：

```python
Solution().twoSum(nums, target)
```

Python 实际会自动把 `Solution()` 这个对象传进去：

```python
Solution.twoSum(Solution(), nums, target)
```

所以方法定义里必须用 `self` 接住这个对象。

## 9. 业界怎么用

业界不会天天纠结 `self / cls / staticmethod`，真正使用时通常按**业务归属**来选。

### 9.1 实例方法：最常用

业界最多用的是实例方法。

```python
class UserService:
    def __init__(self, user_repo):
        self.user_repo = user_repo

    def get_user(self, user_id):
        return self.user_repo.find_by_id(user_id)
```

这里用 `self`，因为这个方法依赖对象自己的属性：

```python
self.user_repo
```

实际项目里常见于：

| 场景 | 例子 |
|---|---|
| Service 层 | `UserService.create_user()` |
| Repository / DAO 层 | `UserRepository.find_by_id()` |
| Client 封装 | `Neo4jClient.query()`、`RedisClient.get()` |
| 模型对象行为 | `order.calculate_total()` |
| 状态管理 | 对象里面有连接、配置、缓存 |

一句话：

> 只要这个方法需要用对象内部状态，就用实例方法。

比如：

```python
class Neo4jClient:
    def __init__(self, driver):
        self.driver = driver

    async def query(self, cypher):
        return await self.driver.execute_query(cypher)
```

这里必须是实例方法，因为 `query` 依赖：

```python
self.driver
```

---

### 9.2 类方法：常用于构造对象

类方法在业界主要用于**工厂方法**，也就是从不同来源创建对象。

比如从字典创建对象：

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    @classmethod
    def from_dict(cls, data):
        return cls(
            name=data["name"],
            age=data["age"]
        )
```

调用：

```python
user = User.from_dict({
    "name": "张三",
    "age": 18
})
```

为什么不用普通方法？

因为对象还没创建。

你是先有：

```python
{"name": "张三", "age": 18}
```

然后才创建：

```python
User(...)
```

所以应该用类方法。

另一个例子：

```python
class Settings:
    @classmethod
    def from_env(cls):
        return cls(
            database_url=os.getenv("DATABASE_URL"),
            redis_url=os.getenv("REDIS_URL")
        )
```

一句话：

> 类方法常用于“还没有对象，但我要根据某些数据创建对象”。

---

### 9.3 静态方法：工具函数，但不要滥用

静态方法常用于这个函数**逻辑上属于这个类**，但它不需要 `self`，也不需要 `cls`。

```python
class PasswordUtils:
    @staticmethod
    def is_strong(password):
        return len(password) >= 8
```

调用：

```python
PasswordUtils.is_strong("abc12345")
```

实际项目里常见于：

| 场景 | 例子 |
|---|---|
| 格式校验 | `is_valid_email()` |
| 数据转换 | `parse_date()` |
| 简单计算 | `calculate_score()` |
| 字符串处理 | `normalize_text()` |
| 规则判断 | `is_legal_query()` |

但业界也很谨慎用静态方法。因为如果一个方法完全不依赖类，那它可能更适合写成普通函数：

```python
def is_valid_email(email):
    ...
```

而不是硬塞进类里：

```python
class EmailUtils:
    @staticmethod
    def is_valid_email(email):
        ...
```

这种 `Utils` 类太多，项目就会变成工具箱坟场。

## 10. 怎么判断用哪种方法

| 判断问题 | 使用方法 |
|---|---|
| 要操作对象自己的数据，比如 `self.name` | 实例方法 |
| 要操作类变量，比如 `cls.count` | 类方法 |
| 要根据配置、字典、字符串创建对象 | 类方法 |
| 不需要对象，也不需要类，只处理参数 | 静态方法或普通函数 |
| 需要数据库连接、缓存、配置等对象状态 | 实例方法 |

更短：

```text
用对象数据 -> self
用类数据 -> cls
只做工具 -> staticmethod / 普通函数
```

## 11. 面试回答

Python 类中常见三种方法：实例方法、类方法和静态方法。

实例方法第一个参数是 `self`，代表当前对象，主要用于访问或修改对象自己的状态，比如对象里的数据库连接、配置、缓存、repository 等。

类方法通过 `@classmethod` 定义，第一个参数是 `cls`，代表当前类，常用于操作类变量，或者作为工厂方法创建对象，比如 `from_dict()`、`from_config()`、`from_env()`。

静态方法通过 `@staticmethod` 定义，不接收 `self` 或 `cls`，它只是逻辑上放在类里的工具函数，比如格式校验、数据转换、简单计算。但如果这个函数和类关系不强，直接写普通函数会更清晰。

在实际项目中，最常用的是实例方法，因为业务代码通常依赖对象内部状态。类方法更多用于构造对象或处理类级别逻辑。静态方法适合工具逻辑，但要避免把项目写成大量没有业务含义的 `Utils` 类。

一句话总结：

```text
实例方法处理对象状态；
类方法处理类级别逻辑和对象构造；
静态方法只是放在类命名空间下的工具函数。
```
